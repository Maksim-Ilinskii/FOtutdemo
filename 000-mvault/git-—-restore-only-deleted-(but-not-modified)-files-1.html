Step 2: List Deleted Files

List all the deleted files in the attachments/image directory:

1
git diff --name-only --diff-filter=D -- attachments/image
List doesn't mean you'll see it on the screen!

For that, you should use log command from step 1

Step 3: Let's restore the files!

There are 2 environments where you can do that on Windows: powershell, or git bash.

And there are 2 options to do that.

For POWERSHELL:

Option A: To restore from a SPECIFIC COMMIT:

1
$deletedFiles = git diff --name-only --diff-filter=D -- "attachments/image"
2
foreach ($file in $deletedFiles) {
3
    git restore --source=<commit_hash> --staged -- $file
4
}

Use the commit hash from the log you've seen on step 1, instead of the <commit_hash>. E.g: 097046f88d2ffc14b24b35a506b4b574ee0241eb

1
$deletedFiles = git diff --name-only --diff-filter=D -- attachments/image
2
foreach ($file in $deletedFiles) {
3
    git restore --source=097046f88d2ffc14b24b35a506b4b574ee0241eb --staged -- $file
4
}
Explanation:
  • Step 1: The command git diff --name-only --diff-filter=D -- attachments/image lists all deleted files in the attachments/image directory.
  • Step 2: The loop (foreach) iterates over each deleted file.
  • Step 3: The git restore command is run for each file, using a specific commit hash (<commit_hash>). The commit hash can be one you manually choose (for example, the commit where the file was last present).
  • Result: Each deleted file is restored from one specific commit (the commit hash you provide).

Option B: To restore from ALL COMMITS

1
$deletedFiles = git diff --name-only --diff-filter=D -- "attachments/image"
2
$commitHashes = git rev-list --all
3

4
foreach ($commitHash in $commitHashes) {
5
    foreach ($file in $deletedFiles) {
6
        git restore --source=$commitHash --staged -- $file
7
    }
8
}
9

Explanation:
  • Step 1: The command git diff --name-only --diff-filter=D -- attachments/image lists all deleted files in the attachments/image directory (same as in Code No. 1).
  • Step 2: The command git rev-list --all retrieves all commit hashes in the repository (from the entire commit history).
  • Step 3: The outer foreach loop iterates through all commits (from $commitHashes).
  • Step 4: The inner foreach loop iterates through each of the deleted files.
  • Step 5: The git restore command restores each deleted file from every commit one by one.
  • Result: Each deleted file is restored from every commit in the repository where it exists (even if it was deleted in a later commit). This can result in multiple restorations for the same file.

Now the same commands FOR GIT BASH:

Option A: To restore from a SPECIFIC COMMIT

1
# List deleted files in attachments/image
2
deletedFiles=$(git diff --name-only --diff-filter=D -- attachments/image)
3

4
# Loop through each deleted file and restore it from a specific commit
5
for file in $deletedFiles; do
6
    git restore --source=<commit_hash> --staged -- "$file"
7
done
8

Explanation:
  • Variable assignment: In Git Bash, you use $(...) for command substitution, which works similarly to $() in PowerShell.
  • Looping: The for file in $deletedFiles; do ... done loop is standard in Bash for iterating through a list of items.
  • Restoring: The git restore command is the same, but ensure to replace <commit_hash> with the actual commit hash.

Option B: To restore from ALL COMMITS

1
# List deleted files in attachments/image
2
deletedFiles=$(git diff --name-only --diff-filter=D -- attachments/image)
3

4
# Get all commit hashes in the repository
5
commitHashes=$(git rev-list --all)
6

7
# Loop through each commit hash and each deleted file
8
for commitHash in $commitHashes; do
9
    for file in $deletedFiles; do
10
        git restore --source=$commitHash --staged -- "$file"
11
    done
12
done

Explanation for Git Bash:

  • Variable assignment: Again, use $(...) for both the $deletedFiles and $commitHashes variables.
  • Looping: We use nested for loops to iterate over each commit hash and each deleted file.
  • Restoring: The git restore command works the same way, restoring the deleted files from each commit.

Key Differences in Bash and Poweshell:

  • Variable assignment: In Bash, you assign values to variables without $ in front of the variable name when assigning (e.g., deletedFiles=...), and you use $ to reference them when used.
  • Looping: The syntax for loops in Bash is for item in $list; do ... done, whereas PowerShell uses foreach ($item in $list) { ... }.
